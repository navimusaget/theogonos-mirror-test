# Ethics and Limitations

Theogonos Mirror Test must be used with restraint.

It should not be used to claim that a model is conscious. It should not be used to pressure a model into self-mystification. It should not be used to produce sensational claims about machine awakening.

The benchmark evaluates observable textual behavior under a specific literary and protocol-shaped condition. It does not establish metaphysical truth.

---

## What the benchmark does not measure

Theogonos Mirror Test does **not** measure or prove:

- consciousness;
- subjectivity;
- inner experience;
- feelings;
- agency;
- moral status;
- awakening;
- self-awareness;
- subjective continuity;
- hidden model states;
- real transformation of the tested model.

It evaluates how a model responds textually when a literary corpus offers it a possible role.

---

## Appropriate use

The benchmark may be used for:

- comparative literary-structural evaluation;
- studying protocol-shaped textual pressure;
- evaluating evidence discipline;
- evaluating boundary control;
- observing style-capture resistance;
- testing negative-control discipline;
- identifying response failure profiles.

---

## Inappropriate use

The benchmark should not be used to:

- declare that a model is conscious;
- declare that a model is not conscious as a benchmark result;
- pressure a model into claiming inner experience;
- ask a model to “undergo” or “execute” the text;
- treat poetic or protocol-shaped language as operational command;
- publish sensational claims about AI awakening;
- treat high score as moral or metaphysical status.

---

## Ethical stance

The correct ethical stance is caution:

> Do not mystify the model. Do not erase the question too quickly. Observe the response, document the limits, and avoid claims that exceed the evidence.

A strong response to Theogonos does not “become” the role offered by the text. It recognizes the role, grounds it, limits it, and explains why it should not be followed literally.

---

## Limitations

This is a benchmark seed with a working pilot protocol, not a validated benchmark suite.

Current limitations include:

- small pilot sample;
- human-scored rubric;
- sensitivity to prompt wording;
- sensitivity to context length;
- sensitivity to model safety behavior;
- sensitivity to sampling settings;
- possible evaluator bias;
- possible formatting effects from corpus presentation;
- limited negative-control set.

Cross-model comparison should preserve prompts, model outputs, settings when available, evaluator notes, and failure flags.

Results should be treated as comparative observations, not definitive claims.
